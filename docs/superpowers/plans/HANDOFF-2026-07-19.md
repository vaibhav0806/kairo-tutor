## What's done + verified

### Backend (`server/`) — Fastify + Neon (Drizzle) + Better Auth + AI proxy
- **Auth:** self-hosted Better Auth 1.6.23, Google-only, JWT (15m) + JWKS + bearer. Three owned
  routes for the desktop `kairo://` handshake: `GET /auth/start` → Google, `GET /auth/callback` →
  `kairo://auth-callback?code=…`, `POST /auth/exchange {code}` → `{ sessionToken, expiresAt }`.
  `POST /api/auth/token` (bearer=sessionToken) → JWT; `GET /api/auth/jwks`.
  - **Live-verified:** `/auth/start` returns a real 302 to `accounts.google.com` with PKCE (S256),
    scopes `email profile openid`, redirect `http://localhost:8787/api/auth/callback/google`.
  - `/auth/exchange` creates a session row directly (1.6 has no server `createSession`); proven to
    authenticate `/api/auth/token` and yield a JWT.
- **Metering:** `usage_counter` (used_free / free_limit / plan), atomic
  `UPDATE … WHERE used_free < free_limit RETURNING` reserve + idempotent `usage_event` + refund on
  failure. Metered on **`POST /v1/vision/tutor` only** (= 10 free asks). Verified 200→402.
- **AI proxy (key-injecting, JWT-gated):** `POST /v1/llm/chat` (OpenRouter), `POST /v1/vision/tutor`
  (metered; `_provider:'anthropic'|'openai'` picks the upstream), `POST /v1/vision/point` (OpenAI),
  `POST /v1/stt` (multipart→Sarvam), `POST /v1/tts`, `POST /v1/tts/stream` (PCM passthrough, aborts
  upstream on client disconnect). `GET /v1/me` → `{plan,status,usage,renews_at,paywalled}`.
- **Billing (Dodo):** `applyDodoState()` maps the subscription lifecycle → `subscription` row +
  denormalized `usage_counter.plan` (cancel keeps pro to period end, on_hold 3-day grace, expire→
  free), out-of-order guard, webhook idempotency (`webhook_event`). `POST /webhooks/dodo` (scoped
  raw-body + Standard Webhooks HMAC). `POST /v1/billing/checkout` + `/v1/billing/portal`.
  - **Live-verified (test mode):** created the two Pro products and a real checkout session.

## Remaining — Plan 2: Desktop cutover (Rust + frontend)

Goal: the app authenticates via the browser and routes **all** provider calls through the backend.
**Important:** the backend uses **semantic** routes (not a transparent base-URL passthrough), so
each Rust call site maps to a specific route + sends the same provider body, swapping the provider
key for the session JWT.

**Recommendation:** gate the provider-repoint behind a `USE_BACKEND_PROXY` constant (default
`false`) so the working direct-provider path stays intact until you flip it with the backend running.

### Native (Rust, `src-tauri/`)
1. `Cargo.toml`: add `tauri-plugin-deep-link` (v2) and `keyring` (v3).
2. `tauri.conf.json`: `plugins.deep-link.desktop.schemes = ["kairo"]`. Verify it injects
   `CFBundleURLTypes` into `Info.plist` after a build (add manually if not). **No entitlement needed.**
3. New `src-tauri/src/auth.rs`:
   - Keychain via `keyring` (service `com.kairo.tutor`, items `session` / `refresh`), `get/set/clear`.
   - `start_google_auth()` command → open the system browser at `{KAIRO_BACKEND_URL}/auth/start`
     (reuse the `open` pattern in `permissions.rs`).
   - deep-link handler (registered in `lib.rs` `setup()` via `on_open_url`): parse
     `kairo://auth-callback?code=…` → `POST {backend}/auth/exchange {code}` → store `sessionToken`
     in Keychain → `app.emit("auth:changed", …)`. **Must not** foreground the app (keep Accessory
     policy — don't call `set_activation_policy(Regular)`). **Redact** the code in logs.
   - Token helper: cache a JWT from `POST {backend}/api/auth/token` (bearer=sessionToken), refetch
     on 401. `get_auth_status()` and `sign_out()` commands.
4. `constants.rs`: add `KAIRO_BACKEND_URL` (e.g. `http://localhost:8787`) + `USE_BACKEND_PROXY:bool`.
5. Repoint the 12 provider call sites (behind the flag). Map each to the semantic route, attach
   `Authorization: Bearer <jwt>`, drop the provider key:
   - `run_gate_turn` / `run_ack_turn` (OpenRouter chat) → `POST /v1/llm/chat` (same chat body).
   - `run_tutor_turn` vision → `POST /v1/vision/tutor` with the existing OpenAI-Responses/Anthropic
     body **plus** `_provider` and header `X-Kairo-Ask-Id: <uuid minted per ask>`.
   - `detect_click_point_openai` → `POST /v1/vision/point`.
   - `transcribe_audio` → `POST /v1/stt` (multipart, one `file` field).
   - `synthesize_speech` → `POST /v1/tts`; `synthesize_speech_stream` → `POST /v1/tts/stream`
     (unchanged chunk loop; only URL + bearer change).
   - Centralize in one `backend_request(path)` helper next to `shared_http_client`.
6. Typed errors: map backend `401`→`unauthenticated`, `402`→`quota_exceeded`, network→`offline`, so
   the notch can branch (see frontend below). `cargo check` green before committing.

## Notes / decisions taken during the build

- **Semantic routes, not transparent passthrough** — cleaner metering + key handling. Desktop cutover
  maps per-route (above).
- **`@kairo/shared` is source-only**; the server bundles it via `tsup` (`noExternal`) so prod
  `node dist` is self-contained.
- **Sessions in `/auth/exchange`** are created by direct row insert (no server `createSession` in
  1.6). Verified to bearer-authenticate.
- **Tests run sequentially** (`fileParallelism:false`) — they share one Neon DB and some listen on
  8787 for the JWKS fetch.
- **Neon SSL:** `rejectUnauthorized:false` (common Neon dev setting); tighten for prod if desired.
  A pg v9 deprecation warning about `sslmode` aliases is harmless for now.
- **Security:** no secrets committed; `server/.env` gitignored (verified). Dodo operated in **test
  mode only**. Live Dodo key still sits in root `.env` — remove it when convenient (per your note).
